# Capstone

1. 目標與需求
    - 開發一個更聚焦、更標準的工具，以適應當前集中化的工作趨勢，提升整體效率
    - 取代目前依賴IDEA工具的做法，以降低成本並增加彈性
    - 提供更優化的審計流程，減少人工作業時間，提高準確性
    - 透過工具優化，讓服務時間縮短或品質提升，進一步提升事務所的競爭力
2. 開發策略與技術選擇
    - 現有工具 (如 IDEA) 雖然可以處理大數據，但有高昂成本與技術依賴問題
    - 考慮使用 Excel + VBA 及 Access 來開發新工具
        - 可處理百萬筆資料，符合目前 80% 以上的案件需求
        - 限制:Access 過6萬筆資料可能變慢，1000萬筆以上IDEA仍需支援
    - 避免依賴 SQL 及外部環境，減少 IT 部門的負擔
    - 開發目標功能
        - 自動驗證
        - 會計檢查
        - 常見條件檢查
        - 標準化輸出文件
        - 考量未來擴展性，如整合AI輔助分析、適應更大資料量的架構
3. 專案執行計畫
    - 學生參與角色
        - 參與測試、回饋工具使用體驗
        - 建立虛擬個案，測試新工具能否有效發現錯誤
    - 課程策略
        - 給予學生「最低限度可行版本(MVP)」讓其測試並給予回饋
        - 透過分組討論，確保理解工具功能與實際應用
4. 可能面臨的挑戰
    - 學生技術背景參差不齊，可能影響開發進度
    - 資料量限制:Access可能無法支撐大數據需求，需進一步確認IT部門意見
    - 期末報告與成果驗證:
        - 報告內容須包含:
            - 工具開發進度與成效
            - 具體數據顯示如何提升效率(如工時降低、錯誤減少)
            - 學生的回饋與測試結果
        - 測試結果影響最終工具設計，須提前測試，避免臨時修正影響專案完整性
5. 預期成果
    - 成功開發 Excel/Access-based 工具，降低對 IDEA 的依賴
    - 透過學生參與，驗證工具的可行性與實際應用成效
    - 期末報告提供完整數據分析，展示效率提升與潛在成本節省
    - 建立標準化的工具開發框架，未來可應用於其他專案
6. 後續行動
    - 確認IT部門對Access/SQL的支援
    - 與KCT確認目前資料處理限制與可行性
    - 與學生進行初步技術說明，確保開發計畫順利推行
